## Why

長輩（退休會計）每月手動整理多家銀行基金對帳單至 Excel，目前只能透過大量數字表格來檢視投資狀況。缺乏視覺化工具讓長輩難以快速掌握總資產走勢與個別基金績效，也不方便在手機上瀏覽。

## What Changes

- 新增一個 Web Dashboard，從 Excel 月度快照資料產生互動式走勢圖
- 支援多維度切換（銀行、基金、憑證、幣別、配息類型、季度、年度）
- 報酬率支援「含息」與「不含息」兩種顯示模式（類似股票現價 vs 還原價）
- 提供總資產 Overview 與個別基金深入兩種瀏覽模式
- 響應式設計，支援手機與電腦操作
- 介面採繁體中文，字體放大，適合長輩使用

## Capabilities

### New Capabilities

- `data-parser`: 解析 Excel 月度快照 sheet，正規化為統一 JSON 結構（處理銀行分群、匯率、欄位差異）
- `asset-overview`: 總資產 NTD 月度走勢圖 + 各銀行/幣別佔比圓餅圖
- `bank-cash-flow`: 各銀行帳戶配息現金流（流進資產）狀況走勢圖，用於追蹤轉投資資金來源
- `fund-detail`: 單一基金或憑證的現值、損益、配息走勢圖，支援「含息報酬」與「不含息報酬」切換（類似股票現價 vs 還原價），並透過視覺化呈現漲跌趨勢（取代原本手動標記的 DU 欄位）
- `dimension-filter`: 多維度篩選器，支援以下維度：
  - 銀行（兆豐、玉山、合庫等）
  - 基金名稱
  - 憑證號碼（同名基金多筆投資區分）
  - 幣別（USD / NTD / CNY）
  - 配息類型（有配息 / 無配息）
  - 時間範圍（季度、年度）

### Modified Capabilities

（無，此為全新專案）

## Impact

- 新增 `src/` 目錄：HTML + CSS + JavaScript 檔案
- 新增 `data/processed/`: 正規化後的 JSON 資料
- 依賴：Chart.js 或類似圖表庫（CDN 引入，無需 npm）

## 長輩每月動態輸入欄位

以下兩個欄位是長輩每月從銀行對帳單手動填入的核心資料，**無法從其他欄位推算**：

| 欄位 | 說明 |
|------|------|
| 現值 | 基金當月市值，依銀行對帳單而定 |
| 累計配息 | 歷史累計配息金額，配息金額會受現值高低影響 |

其他欄位（如含息損益、不含息損益、含息報酬%）均可由上述兩個欄位搭配成本等靜態資料推算而得。

## 非目標

- 不處理 OCR 或自動對帳單解析
- 不串接 Google Sheets API（本階段使用靜態 Excel 資料）
- 不處理合股、房貸等非銀行投資產品
- 不理會原有資料中的現值 DU 欄位（過去手動標記錯漏多，改由系統比對前後期數字自動呈現漲跌）
- 不建立使用者登入或權限機制
